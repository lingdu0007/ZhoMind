# ZhoMind 前端极简重构设计规范（2026-04-13）

## 1. 背景与目标
- 前后端分离，后端 FastAPI，前端静态文件，通过 Nginx 部署；所有模型调用走硅基流动 API。
- 目标：为 Agentic RAG 控制台构建桌面端（≥1280px）极简体验，保持功能完整的同时在视觉上体现 Exaggerated Minimalism。
- 输出：12 列网格布局方案 + 聊天、文档上传、配置页的极简交互与视觉规则。

## 2. 全局布局与导航
- **网格**：最大宽度 1440px，12 列，每列间距 24px。侧栏占 2 列（≈200px），主内容占 8 列（≈960px），右侧保留 2 列呼吸区。
- **侧栏**：背景 `#F1F5F9`，与主区之间使用 `1px #E2E8F0` 分割线。内容仅包含 Logo + “ZhoMind”、导航列表（聊天/文档/配置）。Lucide 20px 线性图标，hover 背景 `#E2E8F0`，保持纯导航不过载状态信息。
- **顶部控制条**：位于主内容顶部，左侧显示页面标题（如 “Chat · Knowledge Pulse”），右侧包含“会话”按钮（抽屉入口）、设置、头像。背景白色，底部细线 `#E2E8F0`。
- **页面容器**：所有主内容（聊天流、上传面板、配置卡片）都在 8 列区域内渲染；上传/配置页面可让单个卡片跨 6 列以实现 6+6 排列。

## 3. 聊天体验
- **会话抽屉**：顶栏右侧按钮触发，全宽 8 列，位置位于输入区上方。容器圆角 12px，阴影 `0 20px 60px rgba(15,23,42,0.1)`。内容展示最近 20 个会话，列表项包含会话名 + 更新时间 + 消息数，hover 背景 `#F1F5F9`。删除按钮用幽灵样式，刷新按钮放在抽屉右上角。
- **消息流**：单列垂直对齐。每条消息使用 1px 细边框 + 2px 阴影，背景白色，内边距 24px。系统/Agent 在左（左侧竖线 `#475569`），用户在右（竖线 `#64748B`），但整体仍保持统一宽度以保证整洁。错误提示直接以红色条展示，遵守 Let it crash。
- **输入区**：固定在视窗底部，宽 8 列，背景白色浮层，阴影 `0 8px 30px rgba(15,23,42,0.06)`。多行文本框采用上下 1px 线条 + `#F8FAFC` 背景，聚焦线条变为主色 `#475569`。右侧按钮：线框“停止”、实心 CTA “发送”（`#2563EB`）。Streaming 状态在顶部标题下方以 12px 文本 + 3 点淡入动效呈现。

## 4. 文档上传页
- **上传卡片**：跨 8 列，白底、圆角 16px、阴影 `0 12px 40px rgba(15,23,42,0.08)`。包含拖拽区域（虚线边框 `#CBD5F5`）、“选择文件”按钮和上传进度条。下方展示错误/成功状态条。
- **统计条**：上传卡片下显示 3 个等宽迷你卡片（总文档、向量块数、上次同步），每个卡片高度 80px，背景 `#FFFFFF`，轻阴影。
- **表格**：Element Plus Table 自定义主题：表头背景 `#F8FAFC`、字体 14px/500、行高 48px；行 hover 色 `#EEF2FF`。列分隔线 `1px #E2E8F0`。操作列按钮使用幽灵样式（文本 + 边框 `#CBD5F5`）。顶部工具条提供搜索、刷新、批量删除。
- **空态**：浅灰线条插画 + 文案 “暂无文档，上传文件以构建知识库”，附 CTA 按钮。

## 5. 配置页
- **卡片布局**：在 12 列网格中每卡片跨 6 列（大屏双列，窄屏自动换行），纵向间距 24px。卡片白底、圆角 16px、阴影 `0 8px 30px rgba(15,23,42,0.06)`。顶部包含标题与副标题说明。
- **卡片类型**：  
  1. **模型配置**：LLM/Embedding/Rerank/Agent 参数。表单两列布局（标签宽 120px），底部提供“重置”“保存”按钮，保存是实心 CTA。  
  2. **检索参数**：Top K、阈值、Dense/Sparse 权重，使用滑块+数字输入联动，底部展示实时比例提示。  
  3. **存储配置**：Milvus/Redis/缓存连接等字段使用等宽字体输入框，右上角显示状态徽标（在线/离线）。  
  4. **安全 & API**：硅基流动 API Key、回调 URL、限流参数。敏感字段默认隐藏，提供显示/复制按钮。
- **交互细节**：每卡片独立 loading/error 状态；保存成功在卡片底部显示淡绿色文本“已保存 · HH:MM”；失败信息在顶部红色条展示。Tab 顺序按字段顺序，聚焦时卡片边框变 `#2563EB`。

## 6. 视觉系统与动效
- **色彩**：背景 `#F8FAFC`、主内容白色；主文本 `#1E293B`，次文本 `#475569`；主色 `#475569`，CTA 蓝 `#2563EB`；警告 `#F97316`、错误 `#DC2626`。所有可点击元素 `cursor: pointer`。
- **字体**：Fira Sans（300/400/600）+ Fira Code（等宽字段）。标题字重 600，正文 400；代码/配置值使用 Fira Code 400。
- **组件统一**：卡片圆角 16px，阴影 `0 8px 30px rgba(15,23,42,0.06)`；按钮高度 44px，线框按钮边框 `#CBD5F5`；输入框高度 44px，内边距 16px。
- **动效**：默认过渡 `transition: all 200ms cubic-bezier(0.4,0,0.2,1)`；会话抽屉使用高度+透明度动画；Streaming 指示使用 3 点淡入；prefers-reduced-motion 时禁用位移/高度动画，仅保留颜色变化。
- **可访问性**：文本对比度 ≥4.5:1；focus ring `2px #2563EB`；所有状态色提供文字描述，不靠颜色区分；空态、错误信息使用文本描述 + icon。

## 7. 实施注意
- 组件库延续当前 Vue + Element Plus，但需重写主题变量与部分样式以符合设计系统。
- Tailwind/自定义 CSS 采用 CSS 变量表示主色/背景/边框，便于后续暗色或品牌扩展。
- 所有交互遵循 Let it crash：错误直接暴露，不做隐式兜底；按钮禁用与 loading 状态必须显式显示。
